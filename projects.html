Raytracing II

Dec. 2024 | C++ / OpenGL (School project)

I built a raytracer in OpenGL. It builds a BVH for any mesh (this bit of code was given by the teacher; I did pretty much everything else). A shader then accumulates each frame to yield better results.

It supports shadows, reflections, transparency, environment maps, and uses the Cook–Torrance specular reflectance model with microfacet distributions and a GGX distribution.

C++ physics simulation

May 2023 | C++ (School project)

For this project I implemented a physics simulation of particles that can be affected by various forces (gravity, Lennard-Jones potential) in 2D or 3D. The user can specify different parameters such as boundary behaviours or kinetic energy conservation. No external libraries were used, so a custom vector class was written to handle the computations.

The goal was to simulate many particles without having to wait too long, especially since the Lennard-Jones potential requires small time steps. The data structure was therefore an important part of this project.

The pictures below have been computed for a simulation of 8,000 particles over almost 400,000 physics steps in 20 minutes on my laptop.

Black Seas game

Feb. 2023 — Apr. 2023 | Rust / OpenGL (School project)

This is a school project for which we chose the subject. With four friends, we decided to make a game using a game engine that I was developing together with one other member of the group.

The goal was to do everything from scratch, using only OpenGL and a few utility libraries. The objective was a multiplayer game in which players control a boat and navigate on an ocean.

Each member had a dedicated part of the project. Paul Colinot was in charge of physics and collisions. Hélène Hassan worked on the ocean, following Tessendorf's paper. Virgile Henry worked with me on the game engine architecture; he also made the 3D models and added the network code required for multiplayer. I worked on terrain generation, lighting, post-processing effects (ray-marched clouds, gamma correction, tone mapping, bloom), materials and the dynamic skybox.

Many things could have been done better during this project; however, we only had a limited amount of time. We also tried to keep the game engine independent of the game, which slowed us down somewhat because it was not always easy to access the components we needed.

More details are available in the project report (in French) that we submitted, although due to some submission constraints, parts of our work were left out.

Face deformation

Nov. 2022 | Python (School project)

This project's goal was to modify the mesh of a 3D scan of a face by first approximating it with a Bézier surface.

A cut is first performed to apply a least-squares approximation with constraints around the borders, in order to compute a grid of control points for our Bézier surface. The control points can then be easily modified to obtain this final result. The approximated surface can finally be knitted back to the original mesh with a screened Poisson surface reconstruction.

Raytracer in Rust

Aug. 2022 | Rust

A basic CPU raytracer I worked on to learn Rust, following the "Raytracing in One Weekend" series.

This raytracer can handle spheres, depth of field and three types of material:

  • Lambertian
  • Dielectric
  • Metallic

I decided to stop at this point because I wanted to make a GPU-accelerated version of the project later.

While computing the output image, the current state is displayed in a window managed on a different thread.
