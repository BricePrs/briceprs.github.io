C++ Physics Simulation

May. 2023 | C++ (School Project)

For this project I implemented a physic simulation of particles that can be affected by some forces (gravity, lennard-jonnes potential) in 2D or 3D. The user can specify different parameters such as boundary behaviours or kinetic energy conservation. No libraries were used during this project, so a custom Vector class has been made in order to make the computations

The goal was to simulate many particle without having to wait too long especially since lennard-jonnes potential required small-time steps. Therefore data structure was an important part of this project.

The pictures below have been computed for a simulation of 8000 particles over almost 400 000 physics steps in 20 minutes on my laptop

BlackSeas Game

Feb. 2023 - April. 2023 | Rust / OpenGl (School Project)

This project is a school project were we decided the subject. I decided with 4 friends to make a game using a game engine I was making with one other member of this group.

The goal was to do everything from scratch, using only opengl and some other utility libraries. The objective was to make a multiplayer game were player could control a boat and navigate on an ocean.

Each member had a dedicated part for this project. Paul Colinot was in charge of the physics and collisions. Hélène Hassan worked on the ocean, following Tessendorf paper. Virgile Henry was working with me on the game engine architecture. He also made the 3D models and added the network part required for the multiplayer. I worked on the terrain generation, the lighting, postprocessing effects (raymarched clouds, gamma correction, tone mapping, bloom), materials and the dynamic skybox.

Many things could have been done better during this project, however we only had a limited amount of time to spend on this. Finally, I want to explain that we tried to keep the game engine independent of the game, and it has also slowed us down, because it was not always easy to access some of the components we needed.

More details are available on the project report (in french) that we submitted, even though, due to some submissions constraints, some part of our work have been eluded.

Face deformation

Nov. 2022 | Python (school project)

This project's goal was to modify the mesh of a 3D scan of a face by approximating it first with a Bezier surface.

A cut is first performed to apply a least squared approximation with constraint around the borders to compute a grid of control points for our Bezier surface. Then, the control points can be easily modified in order to get this final result. The approximated surface can then easily be knitted back to the original mesh with a screened Poisson surface reconstruction.

Personal Website

Oct. - Nov. 2022 | html, css, javascript, WebGL

I wasn't familiar with front end programing (nor particularly interested) but I wanted to have a website in order to easily share my projects. So I decided to make this website. The goal here is not to have something perfect but something that works.

I made it in roughly 3 weeks since I had to learn html/css and javascript as well as the specificity of WebGL compared to OpenGl 3. The background animation is entirely made with WebGl. It consists of a vertex shader and a post-process fragment shader.

It is definitely far from being perfect and I guess the code could be cleaner (I'm no javascript expert) but I didn't want to spend too much time on it. The user experience is definitely better on computer (if you are on a mobile, I recommend you to check it out on your computer).

Rustracer

Aug. 2022 | Rust

A basic raytracer implementation(on CPU) I've worked on in order to learn Rust following Raytracing in one weekend series.

This raytracer can handle spheres, depth of field and three types of material:

  • Lambertian
  • Dielectric
  • Metalic
However, I decided to stop at this point because I wanted to make a GPU accelerated version of this project later.

While computing the output image, the current state is displayed in a window managed on a different thread.
